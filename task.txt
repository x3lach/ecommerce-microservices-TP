# Project: E-Commerce Frontend (MVP - Auth & Dashboard)
# Tech Stack: React (Vite), React Router DOM, Axios, Tailwind CSS (optional)

## Context
I have a backend microservices architecture running locally behind an API Gateway.
- Base URL: http://localhost:8081/api/v1
- Auth System: JWT-based. The backend returns a raw String token (not JSON) upon login.
- User Roles: Users can be either 'CLIENT' or 'SELLER'.

## Objective
Create a simple React application with three pages to test the full Authentication flow.

## 1. Project Setup
- Initialize a React project using Vite.
- Install `react-router-dom` for navigation.
- Install `axios` for API calls.
- Install `jwt-decode` to extract user info from the token.

## 2. Authentication Context (`AuthProvider`)
Create a Context to manage the user's global state.
- It should check `localStorage` for an existing token on load.
- Functions needed:
  - `login(token)`: Save token to localStorage, decode it to get User ID/Role, set state, navigate to Dashboard.
  - `logout()`: Remove token from localStorage, clear state, navigate to Login.
  - `user`: State variable holding { email, role, sub (id) }.

## 3. Page Requirements

### A. Register Page (`/register`)
- Fields:
  - Full Name (text)
  - Email (email)
  - Password (password)
  - Role (Dropdown/Select: "CLIENT" or "SELLER")
- API Call: POST http://localhost:8081/api/v1/users
- Body JSON:
  {
    "fullName": "...",
    "email": "...",
    "password": "...",
    "role": "..."
  }
- Behavior: On success, redirect to Login page.

### B. Login Page (`/login`)
- Fields:
  - Email
  - Password
- API Call: POST http://localhost:8081/api/v1/users/auth/login
- Behavior:
  - The backend returns the Token as a plain string.
  - Capture the token, call `login(token)` from Context, redirect to Dashboard.

### C. Dashboard / Main Page (`/`)
- Protection: This route must be protected. If no token exists, redirect to /login.
- Content:
  - Display a welcome message.
  - Display the User's Information decoded from the token:
    - "Logged in as: [Email]"
    - "User ID: [sub]"
    - "Role: [role]"
  - "Logout" Button: Calls `logout()` and redirects to login.

## 4. Routing
- `/login` -> Login Component
- `/register` -> Register Component
- `/` -> Dashboard (Protected Route)

## 5. Styling
- Keep it clean and minimal.
- Center forms on the screen.
- Use clear visual cues for buttons and inputs.
